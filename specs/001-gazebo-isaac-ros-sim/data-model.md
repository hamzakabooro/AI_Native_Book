# Data Model: Gazebo-Isaac-ROS Simulation

## Entities Overview

The Gazebo-Isaac-ROS Simulation system involves several key entities that represent the core concepts of robotics simulation and perception. These entities are used throughout the educational content to describe the various components of digital twin creation, sensor simulation, and robot perception.

## Entity 1: Simulation Environment

**Description**: The virtual world containing objects, physics properties, and environmental conditions for robot simulation

**Attributes**:
- Environment ID (unique identifier)
- Name (human-readable environment name)
- Physics Properties (gravity, friction coefficients, collision properties)
- Objects (list of static and dynamic objects in the environment)
- Light Sources (position, intensity, color of light sources)
- Environment Map (2D/3D representation of the environment)

**Relationships**:
- Contains multiple Robot Models
- Contains multiple Objects
- Associated with multiple Sensor Data streams

## Entity 2: Robot Model

**Description**: The digital representation of a physical robot including kinematic properties, sensors, and physical characteristics

**Attributes**:
- Model ID (unique identifier)
- URDF/SDF Path (file path to the robot model definition)
- Kinematic Properties (joint types, limits, DH parameters)
- Physical Properties (mass, inertial properties, collision geometry)
- Sensor Mounts (attachment points for sensors)
- Actuator Properties (joint motors, transmission types)

**Relationships**:
- Belongs to one Simulation Environment
- Contains multiple Sensors
- Associated with Sensor Data streams
- Connected to ROS 2 topics and services

## Entity 3: Sensor Configuration

**Description**: The configuration parameters for a simulated sensor attached to a robot model

**Attributes**:
- Configuration ID (unique identifier)
- Sensor Type (LiDAR, Depth Camera, IMU, etc.)
- Mount Position (relative to robot coordinate frame)
- Mount Orientation (relative to robot coordinate frame)
- Sensor Parameters (range, resolution, noise characteristics, field of view)
- Output Topic (ROS 2 topic where sensor data is published)

**Relationships**:
- Belongs to one Robot Model
- Generates Sensor Data
- Associated with Sensor Simulation

## Entity 4: Sensor Data

**Description**: The information collected by simulated sensors (LiDAR, Depth Camera, IMU) that mimics real-world sensor outputs

**Attributes**:
- Data ID (unique identifier)
- Timestamp (when the sensor data was generated)
- Sensor Type (LiDAR, Depth Camera, IMU, etc.)
- Data Payload (the actual sensor readings - point cloud, image, acceleration data, etc.)
- Frame ID (coordinate frame reference)
- Quality Metrics (noise levels, accuracy indicators)

**Relationships**:
- Generated by Sensor Configuration
- Belongs to Robot Model
- Produced in Simulation Environment
- Used in Perception Pipeline

## Entity 5: Perception Pipeline

**Description**: The processing chain that transforms raw sensor data into meaningful information for robot decision-making

**Attributes**:
- Pipeline ID (unique identifier)
- Pipeline Name (human-readable name)
- Input Topics (ROS 2 topics for sensor data inputs)
- Processing Nodes (sequence of processing nodes in the pipeline)
- Output Topics (ROS 2 topics for processed information)
- Computational Requirements (CPU/GPU usage, memory requirements)

**Relationships**:
- Processes Sensor Data
- Associated with Robot Model
- Uses Isaac ROS components (if applicable)
- Generates Navigation Graphs (in navigation-focused pipelines)

## Entity 6: Navigation Graph

**Description**: The representation of navigable space and pathways used by Nav2 for path planning

**Attributes**:
- Graph ID (unique identifier)
- Map Reference (reference to the environment map)
- Waypoints (coordinates of navigable locations)
- Connections (relationships between waypoints)
- Path Constraints (minimum turning radius, obstacle clearance, etc.)

**Relationships**:
- Associated with Simulation Environment
- Belongs to Robot Model (navigation is robot-specific)
- Used by Nav2 Path Planning

## Entity 7: Simulation Asset

**Description**: Reusable components like 3D models, textures, physics properties, and world configurations

**Attributes**:
- Asset ID (unique identifier)
- Asset Type (model, texture, world, material)
- File Path (location of the asset file)
- Format (URDF, SDF, OBJ, FBX, etc.)
- Dependencies (other assets this asset depends on)

**Relationships**:
- Used in Simulation Environments
- Referenced by Robot Models
- Included in Sensor Simulations

## Validation Rules

1. **Simulation Environment** must have valid physics properties (gravity cannot be negative)
2. **Robot Model** must have a valid URDF/SDF file that passes ROS 2 model validation
3. **Sensor Configuration** parameters must be within realistic ranges for the sensor type
4. **Sensor Data** timestamps must be sequential and within simulation time bounds
5. **Perception Pipeline** must have valid ROS 2 topic connections
6. **Navigation Graph** waypoints must be within the bounds of the referenced map
7. **Simulation Asset** file paths must exist and be accessible

## State Transitions

1. **Simulation Environment**:
   - Draft → Validated → Active → Archived
   - Validation includes checking that all referenced models and assets are accessible

2. **Robot Model**:
   - Design → Simulation-Ready → Validated → Deployed
   - Validation includes URDF/SDF parsing and collision detection testing

3. **Perception Pipeline**:
   - Design → Configuration → Testing → Production
   - Testing includes validation against sample sensor data
